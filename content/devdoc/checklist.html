Release Checklist

For each new release the following tasks should be performed.

Code Quality

  • Check for compiler warnings.
  • Check for missing Javadoc.
  • Check for missing NLS tags.
  • Run JUnit tests.
  • Run findbugs.

Documentation

  • Adjust user documentation to new features, update included screen shots.
  • Finalize change log.

Delivery

  • Adjust plug-in's version numbers.
  • Add new version to the update site definition (site.xml).
  • Create SVN tag.
  • Export EclEmma feature to com.mountainminds.eclemma.releng.package/source.
  • Run com.mountainminds.eclemma.releng.package/build.xml to sign JARs and package the all-in download file eclemma-x.y.z.zip.
  • Upload plug-in and feature JARs to Amazon S3 bucket.
  • Upload all-in download ZIP to SourceForge file release system.
  • Adjust default SF download to new eclemma-x.y.z.zip file.
  • Adjust version in page template for www.eclemma.org.
  • Publish www.eclemma.org.
  • Create SVN repository backup.

Support

  • Close bugs and feature requests resolved by the new release.
  • Add new group for the released version in the bug tracker.
  • Update version information on marketplace.eclipse.org.
