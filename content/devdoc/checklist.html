Release Checklist

For each new release the following tasks should be performed.

Code Quality

  • Check for compiler warnings.
  • Check for missing Javadoc.
  • Check for missing NLS tags.
  • Run JUnit tests.
  • Run findbugs.

Documentation

  • Adjust user documentation to new features, update included screen shots.
  • Update todo list.
  • Finalize change log.

Delivery

  • Adjust plugin's version numbers.
  • Create SVN tag.
  • Build and add result to SF downloads.
  • Create SVN repository backup

Support

  • Add new group for the released version in the bug tracker.
